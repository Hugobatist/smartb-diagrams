  <name>Task 1: Create event-bus.js and diagram-dom.js infrastructure modules</name>
  <files>static/event-bus.js, static/diagram-dom.js</files>
  <action>
Create `static/event-bus.js` as an IIFE exporting `window.SmartBEventBus`:
- Use native `EventTarget` + `CustomEvent` as backbone
- Methods: `on(event, handler)`, `off(event, handler)`, `emit(event, data)`, `once(event, handler)`
- The `on` handler should unwrap `e.detail` so consumers get clean data
- The `off` must support removing the same handler reference (store wrapped handlers in a WeakMap keyed by original handler)
- Keep it under 50 lines

Create `static/diagram-dom.js` as an IIFE exporting `window.DiagramDOM`:
- `getSVG()` — returns `document.querySelector('#preview svg')`
- `findNodeElement(nodeId)` — iterates `[id]` elements matching `/^flowchart-(.+)-\d+$/`
- `findSubgraphElement(subgraphId)` — matches `/^subGraph\d+-(.+)-\d+$/`
- `extractNodeId(element)` — walks up DOM finding node/edge/subgraph identity (consolidates duplicated logic from annotations.js, collapse-ui.js, search.js, diagram-editor.js)
- `getNodeBBox(nodeId)` — returns `getBBox()` of found element
- `getNodeLabel(nodeId)` — returns `.nodeLabel` textContent within node
- `getAllNodeLabels()` — returns all `.nodeLabel` elements from SVG
- `findMatchParent(element)` — walks up to find `.node` or `.cluster` parent
- `highlightNode(nodeId, on)` — adds/removes outline styling
- `getViewBox()` — returns SVG viewBox baseVal
- Keep it under 200 lines
- IMPORTANT: Never cache SVG element references — always re-query, because Mermaid's `render()` replaces the entire SVG via innerHTML

Both files must follow the IIFE pattern: `(function() { 'use strict'; ... window.X = X; })();`
