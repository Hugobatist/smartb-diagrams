Shared state that file-tree needs:
- `currentFile` — the currently selected file path. This is read/written by multiple modules. Move it INTO file-tree.js and expose via `SmartBFileTree.getCurrentFile()` and `SmartBFileTree.setCurrentFile(path)`. Also set `window.currentFile` for backward compat with modules that read it.
- `lastContent` — last rendered content. Also move here and expose via `SmartBFileTree.getLastContent()` / `SmartBFileTree.setLastContent(v)`.

Emit events:
- `SmartBEventBus.emit('file:selected', { path, content })` when loadFile succeeds
- `SmartBEventBus.emit('file:saved', { path })` when saveCurrentFile succeeds

Exposed API:
```javascript
window.SmartBFileTree = {
    refreshFileList,
    loadFile,
    syncFile,
    getCurrentFile: () => currentFile,
    setCurrentFile: (p) => { currentFile = p; },
    getLastContent: () => lastContent,
    setLastContent: (v) => { lastContent = v; },
    createNewFile,
    createNewFolder,
    saveCurrentFile,
    deleteFile,
    renameFile,
};
// Backward compat
window.currentFile = currentFile; // Note: this is a snapshot; update it in setCurrentFile
window.refreshFileList = refreshFileList;
window.saveCurrentFile = saveCurrentFile;
```

NOTE: For `window.currentFile` backward compat, update it every time `setCurrentFile` is called. This handles the case where external modules read `window.currentFile` directly.
