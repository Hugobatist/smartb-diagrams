**Create `static/pan-zoom.js`** as an IIFE exporting `window.SmartBPanZoom`. Extract from live.html:

Pan/zoom state and functions (lines ~942-1018):
- State: `zoom`, `panX`, `panY`, `isPanning`, `panStartX`, `panStartY`, `panStartPanX`, `panStartPanY`, `isInitialRender`
- `applyTransform()` — applies CSS transform to `#preview`
- `zoomIn()`, `zoomOut()`, `zoomFit()` — zoom control functions
- Wheel event handler on `#preview-container` for scroll-to-zoom
- Mousedown/mousemove/mouseup handlers on `#preview-container` for drag-to-pan
- Update `#zoomLabel` text on zoom change

Exposed API:
```javascript
window.SmartBPanZoom = {
    getZoom: () => zoom,
    getPan: () => ({ panX, panY, zoom }),
    setPan: (px, py) => { panX = px; panY = py; applyTransform(); },
    applyTransform,
    zoomIn,
    zoomOut,
    zoomFit,
    setInitialRender: (v) => { isInitialRender = v; },
};
// Backward compat
window.zoomIn = zoomIn;
window.zoomOut = zoomOut;
window.zoomFit = zoomFit;
window.applyTransform = applyTransform;
```

IMPORTANT: `render()` in renderer.js calls `zoomFit()` on initial render. After extraction, renderer.js must call `window.zoomFit()` or `SmartBPanZoom.zoomFit()`. Since both load synchronously and renderer.js loads before pan-zoom.js, the render function only runs AFTER all scripts are loaded (it's called from the init IIFE). So this is safe.

IMPORTANT: The `isInitialRender` flag is used in `render()` to decide whether to auto-fit. After extraction, renderer.js needs to read/write this. Export `getInitialRender()`/`setInitialRender()` and have renderer.js use `SmartBPanZoom.setInitialRender(false)` after first render. Alternatively, keep `isInitialRender` in renderer.js since it's render-related state. Choose whichever keeps the code cleaner — the key is that it works.
