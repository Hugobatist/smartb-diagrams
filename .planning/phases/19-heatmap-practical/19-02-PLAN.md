# Plan 19-02: Real-time Session Updates, Mode Toggle, Empty State

<objective>
During active MCP session recordings, heatmap updates in real-time as record_step is called (not only on end_session). A visible UI control lets users manually switch between risk and frequency modes. Switching files properly re-fetches and applies heatmap data. When no heatmap data exists, a helpful empty state message guides the user.
</objective>

<tasks>

## Task 1: Broadcast heatmap:update on record_step

**File:** src/mcp/session-tools.ts

In the record_step tool handler, after `await sessionStore.recordStep(sessionId, event)`:
1. If the event type is `node:visited` and wsManager is available:
   - Get the diagramFile from `sessionStore.getActiveSession(sessionId)?.diagramFile`
   - Broadcast `heatmap:update` with `{ file: diagramFile, data: { [nodeId]: 1 } }` (incremental delta)
2. This enables real-time heatmap updates without waiting for end_session

**Commit:** feat(19-02): broadcast heatmap:update on each record_step

## Task 2: Add incremental merge to ws-handler.js

**File:** static/ws-handler.js

Update the `heatmap:update` case:
- Instead of replacing visit counts entirely with `SmartBHeatmap.updateVisitCounts(msg.data)`:
  - Check if the current file matches msg.file
  - If the data is a small delta (1-2 keys), call a new `SmartBHeatmap.mergeVisitCounts(msg.data)` method
  - Otherwise (full refresh), continue calling `updateVisitCounts`

Actually, simpler approach: Always merge. Add `mergeVisitCounts` to heatmap.js that adds incoming counts to existing counts.

**Commit:** feat(19-02): merge incremental heatmap updates in ws-handler

## Task 3: Add mode toggle button and empty state to heatmap.js

**File:** static/heatmap.js

1. Add `mergeVisitCounts(delta)` function:
   - For each key in delta, add to existing `state.visitCounts[key]`
   - If heatmap is active and in frequency mode, re-apply

2. Modify `toggle()`:
   - When activating, if both risks and visitCounts are empty, show empty state
   - When activating, default to frequency if visitCounts exist, risk if risks exist, empty otherwise

3. Add `createModeToggle()` function:
   - Creates a small cycle button in the heatmap legend
   - Label shows current mode ("Risk" / "Frequency")
   - Click cycles to the other mode
   - Calls `setMode(newMode)` and re-renders legend

4. Update `updateLegend()`:
   - Add mode toggle button at the top of the legend
   - Show empty state message when no data: "Click nodes to build a frequency map, or use MCP risk annotations"

5. Add `getState()` to public API for testing

**Commit:** feat(19-02): add heatmap mode toggle and empty state UI
