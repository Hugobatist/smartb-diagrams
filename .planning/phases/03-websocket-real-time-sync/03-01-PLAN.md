       - `broadcast(message: object)` method: JSON.stringify, iterate `wss.clients`, send to each with `readyState === WebSocket.OPEN`

         ```typescript
         export type WsMessage =
           | { type: 'file:changed'; file: string; content: string }
           | { type: 'file:added'; file: string }
           | { type: 'file:removed'; file: string }
           | { type: 'tree:updated'; files: string[] }
           | { type: 'connected' };
         ```
       - Use stderr-only logging (import `log` from `../utils/logger.js`)

    3. Create `src/watcher/file-watcher.ts` with FileWatcher class:
       - Constructor takes `projectDir: string` and three callbacks: `onFileChanged(relativePath)`, `onFileAdded(relativePath)`, `onFileRemoved(relativePath)`
       - Uses `chokidar.watch(projectDir, { ... })` with:
         - `ignored` callback: return true for `node_modules`/`.git` directories (check `path.basename`), return true for files not ending in `.mmd`, return false for directories (to traverse into them)
         - `persistent: true`
         - `ignoreInitial: true` (don't fire for existing files)
         - `atomic: true` (handle editor atomic saves)
       - Private `handleEvent(event, filePath)` method: normalizes path using `path.relative(projectDir, filePath).split(path.sep).join('/')`, routes to appropriate callback
       - Listen to `change`, `add`, `unlink` events
       - `async close()` method: calls `await this.watcher.close()`
       - Use stderr-only logging
