    1. Refactor `src/server/websocket.ts` - WebSocketManager for multi-project:

       Replace the single WebSocketServer (`{ server, path: '/ws' }`) with noServer mode:
       - Store a `Map<string, WebSocketServer>` keyed by project name (the namespace)
       - Constructor takes `http.Server` and installs an `upgrade` event handler on it:
         ```typescript
         httpServer.on('upgrade', (request, socket, head) => {
           const { pathname } = new URL(request.url!, `http://${request.headers.host}`);
           // Accept both /ws (default project) and /ws/project-name
           let projectName = 'default';
           if (pathname.startsWith('/ws/')) {
             projectName = decodeURIComponent(pathname.slice(4)) || 'default';
           } else if (pathname !== '/ws') {
             socket.destroy();
             return;
           }
           const wss = this.getOrCreateNamespace(projectName);
           wss.handleUpgrade(request, socket, head, (ws) => {
             wss.emit('connection', ws, request);
           });
         });
         ```
       - `getOrCreateNamespace(name: string)`: creates a `new WebSocketServer({ noServer: true })`, sets up connection handler (send `{ type: 'connected', project: name }`), stores in map, returns it
       - `addProject(name: string)`: ensures namespace exists (calls getOrCreateNamespace)
       - `broadcast(projectName: string, message: object)`: send to all clients in that project's WebSocketServer only
       - `broadcastAll(message: object)`: send to all clients across all namespaces (rarely needed)
       - `close()`: close all WebSocketServer instances in the map
       - Update the `WsMessage` type to include an optional `project` field in the `connected` message:
         ```typescript
         | { type: 'connected'; project: string }
         ```
