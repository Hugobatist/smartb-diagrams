**static/heatmap.js** (NEW, ~200 lines):
- IIFE pattern: `var SmartBHeatmap = (function() { ... })();`
- Internal state: `{ active: false, mode: 'risk', risks: new Map(), visitCounts: {}, savedFills: new Map() }`
  - `mode` can be `'risk'` (from @risk annotations) or `'frequency'` (from session data)

- **Risk colors:**
  ```
  var RISK_COLORS = {
    high:   { fill: '#ef4444', opacity: 0.75 },
    medium: { fill: '#eab308', opacity: 0.60 },
    low:    { fill: '#22c55e', opacity: 0.50 },
  };
  ```

- **intensityToColor(t):** Map 0..1 to cold blue -> hot red gradient.
  ```
  r = Math.round(66 + 189 * t);   // 66 -> 255
  g = Math.round(133 - 53 * t);   // 133 -> 80
  b = Math.round(244 - 244 * t);  // 244 -> 0
  return 'rgb(' + r + ',' + g + ',' + b + ')';
  ```

- **saveFills():** Before applying heatmap, iterate all node elements via `DiagramDOM.getAllNodeElements()`, save their shape fill/fill-opacity to `savedFills` Map.

- **restoreFills():** Restore all saved fills from `savedFills` Map. Clear savedFills.

- **applyRiskOverlay():** For each entry in `risks` Map, find node element via `DiagramDOM.findNodeElement(nodeId)`, find shape child (rect/polygon/circle/path/ellipse), set fill and fill-opacity from RISK_COLORS.

- **applyFrequencyHeatmap(visitCounts):** Find max count. For each nodeId in visitCounts, find node element, find shape child, compute `intensity = count / max`, set fill to `intensityToColor(intensity)`, set fill-opacity to `0.4 + intensity * 0.5`.

- **toggle():** If active, deactivate (restoreFills, set active=false, remove 'active' class from button). If not active, activate (saveFills, apply current mode, set active=true, add 'active' class to button). If both risks and visitCounts are available, prefer frequency mode over risk mode (frequency is more dynamic/useful).

- **updateRisks(risksMap):** Store risks. If active and mode is 'risk', re-apply overlay.

- **updateVisitCounts(counts):** Store visitCounts. If active, switch to frequency mode and re-apply.

- **setMode(newMode):** Set mode, if active, restoreFills + saveFills + re-apply.

- **init():** Set up. Fetch initial heatmap data from `GET /api/heatmap/{currentFile}` on file load.

- **Public API:** `{ init, toggle, updateRisks, updateVisitCounts, setMode, isActive: function() { return state.active; } }`

- Subscribe to `diagram:rendered` event via SmartBEventBus to re-apply heatmap after re-renders (same pattern as breakpoints.js and ghost-paths.js).

**static/heatmap.css** (NEW, ~60 lines):
- `#btnHeatmap.active` style (same pattern as #btnGhostPaths.active from breakpoints.css)
- `.heatmap-legend` container at bottom-right of preview area with color scale indicator
- `.heatmap-legend-item` for risk level labels (High/Medium/Low with color dots)
- `.heatmap-legend-gradient` for frequency scale (blue to red gradient bar)
- Legend is only visible when heatmap is active
