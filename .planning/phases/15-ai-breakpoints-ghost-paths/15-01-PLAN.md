**ghost-store.ts (NEW):** Create `src/server/ghost-store.ts`:
```typescript
import type { GhostPath } from '../diagram/types.js';

export class GhostPathStore {
  private paths = new Map<string, GhostPath[]>();

  add(filePath: string, ghost: GhostPath): void {
    const list = this.paths.get(filePath) ?? [];
    list.push(ghost);
    this.paths.set(filePath, list);
  }

  get(filePath: string): GhostPath[] {
    return this.paths.get(filePath) ?? [];
  }

  clear(filePath: string): void {
    this.paths.delete(filePath);
  }

  clearAll(): void {
    this.paths.clear();
  }
}
```

**websocket.ts:** Extend the `WsMessage` union type with three new variants:
- `{ type: 'breakpoint:hit'; file: string; nodeId: string }`
- `{ type: 'breakpoint:continue'; file: string; nodeId: string }`
- `{ type: 'ghost:update'; file: string; ghostPaths: Array<{ fromNodeId: string; toNodeId: string; label?: string }> }`

**routes.ts:** Add 4 new route handlers. The routes.ts file is at 404 lines — the additions should keep it under 500 lines. Register routes inside the existing `registerRoutes()` function:

1. `GET /api/breakpoints/:file` — Calls `service.getBreakpoints(file)`, returns `{ breakpoints: string[] }` (Set converted to array). Pattern: `^/api/breakpoints/(?<file>.+)$` (method GET).

2. `POST /api/breakpoints/:file` — Reads JSON body `{ nodeId: string, action: 'set' | 'remove' }`. If action=set, calls `service.setBreakpoint(file, nodeId)`. If action=remove, calls `service.removeBreakpoint(file, nodeId)`. Broadcasts `breakpoint:hit` WS message when action=set (so browser shows indicator). Returns `{ ok: true }`.

3. `POST /api/breakpoints/:file/continue` — Reads JSON body `{ nodeId: string }`. Stores a continue signal in a server-level Map<string, boolean> (`breakpointContinueSignals`). The key is `${file}:${nodeId}`. Broadcasts `breakpoint:continue` WS message. Returns `{ ok: true }`.

4. `GET /api/ghost-paths/:file` — Returns `{ ghostPaths: GhostPath[] }` from the ghost store. Pattern: `^/api/ghost-paths/(?<file>.+)$`.

The `breakpointContinueSignals` Map and `ghostStore` instance need to be accessible from routes. **server.ts:** Update `createHttpServer()` to expose `ghostStore` and `breakpointContinueSignals` on the returned ServerInstance interface.
