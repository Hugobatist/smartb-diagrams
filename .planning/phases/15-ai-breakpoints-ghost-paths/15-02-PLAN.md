**tools.ts:** Extend `registerTools()` signature to accept optional dependencies for breakpoint features:
```typescript
export function registerTools(
  server: McpServer,
  service: DiagramService,
  options?: {
    ghostStore?: GhostPathStore;
    wsManager?: WebSocketManager;
    breakpointContinueSignals?: Map<string, boolean>;
  },
): void
```

Add two new tool registrations after the existing 5 tools:

**Tool 6: check_breakpoints**
- Description: "Check if the current node has a breakpoint set. Returns 'pause' if a breakpoint exists and no continue signal is pending, 'continue' otherwise. The AI should respect the pause signal by waiting and re-checking."
- Input: CheckBreakpointsInput
- Handler:
  1. Call `service.getBreakpoints(filePath)`
  2. If nodeId is in breakpoints set:
     a. Check `breakpointContinueSignals` for key `${filePath}:${currentNodeId}`
     b. If continue signal exists: delete the signal (one-time), broadcast `breakpoint:continue` via wsManager, return "continue"
     c. If no continue signal: broadcast `breakpoint:hit` via wsManager, return "pause"
  3. If nodeId is NOT in breakpoints: return "continue"
  4. If `options?.breakpointContinueSignals` is undefined (MCP-only mode without --serve): fall back to just checking breakpoints without signals, always return "pause" or "continue" based on breakpoint presence alone

**Tool 7: record_ghost_path**
- Description: "Record a discarded reasoning branch as a ghost path. Ghost paths are displayed as dashed translucent edges in the browser viewer."
- Input: RecordGhostPathInput
- Handler:
  1. If `options?.ghostStore` exists, call `ghostStore.add(filePath, { fromNodeId, toNodeId, label, timestamp: Date.now() })`
  2. If `options?.wsManager` exists, broadcast `ghost:update` message with all ghost paths for the file
  3. Return success text: `Ghost path recorded: ${fromNodeId} -> ${toNodeId}`
  4. If ghostStore is not available (MCP-only mode), return success text with note: "Ghost path recorded (browser visualization unavailable without --serve mode)"

**server.ts:** Update `createMcpServer()` to accept optional `ghostStore`, `wsManager`, and `breakpointContinueSignals` params. Pass them through to `registerTools()`. In `startMcpServer()`, when `options.serve` is true, pass the `ghostStore`, `wsManager`, and `breakpointContinueSignals` from the createHttpServer result. Update the log message from "5 tools" to "7 tools".
