- `breakpoints: new Set()` — set of nodeIds with active breakpoints (synced from annotations.js)
- `activeNotification: null` — reference to current notification DOM element

Public API:
- `init()` — Subscribe to `diagram:rendered` event via SmartBEventBus to re-apply indicators. - `updateBreakpoints(breakpointSet)` — Update internal state and re-render indicators. Called by annotations.js when it parses new content.
- `applyBreakpointIndicators()` — Query `DiagramDOM.getSVG()`, remove existing `.breakpoint-indicator` elements, iterate breakpoints set, for each: find node element via `DiagramDOM.findNodeElement(nodeId)`, get BBox, create SVG circle at `(bbox.x - 4, bbox.y + bbox.height / 2)` with `r=6`, fill `#ef4444`, class `breakpoint-indicator`. - `showNotification(nodeId)` — Create DOM notification bar (use createElement, not innerHTML for XSS safety). Text: "Breakpoint hit on " + nodeId + ".". Buttons: [Continue] (class `primary`) and [Remove Breakpoint]. Continue calls `continueBreakpoint(nodeId)`, Remove calls `removeBreakpoint(nodeId)`. Prepend to `#preview-container`.
- `hideNotification()` — Remove notification bar DOM element.
- `toggleBreakpoint(nodeId)` — POST to `/api/breakpoints/${currentFile}` with `{ nodeId, action: breakpoints.has(nodeId) ? 'remove' : 'set' }`. Update local state after response.
- `continueBreakpoint(nodeId)` — POST to `/api/breakpoints/${currentFile}/continue` with `{ nodeId }`. Hide notification.
- `removeBreakpoint(nodeId)` — POST to `/api/breakpoints/${currentFile}` with `{ nodeId, action: 'remove' }`. Hide notification. Re-render indicators.
