  <action>
1. **package.json**: Move `picocolors` from `devDependencies` to `dependencies`. Run `npm install` to update lockfile.

2. **src/server/websocket.ts**: Add a `getClientCount(namespace?: string): number` method to the `WebSocketManager` class. If `namespace` is provided, count clients in that specific namespace. If not, count across all namespaces. Only count clients with `readyState === WebSocket.OPEN`.

3. **src/server/routes.ts**: The `registerRoutes` function currently accepts `(service: DiagramService, projectDir: string)`. Add a third optional parameter `wsManager?: WebSocketManager` (import the type). Add a new GET `/api/status` route that returns JSON:
```json
{
  "status": "running",
  "uptime": process.uptime(),
  "port": null,
  "projectDir": projectDir,
  "files": <count from service.listFiles()>,
  "connectedClients": <wsManager?.getClientCount() ?? 0>,
  "activeFlags": [{ "file": "...", "nodeId": "...", "message": "..." }, ...]
}
```
For activeFlags: iterate over `service.listFiles()`, call `service.getFlags(file)` for each, and flatten into a single array with the file name attached.

Note: The `port` field is set to `null` in routes.ts since the route doesn't know the port. The caller (server.ts) can override this if needed, but for the status CLI command, the port is already known from the request URL.

4. **src/server/server.ts**: In `createHttpServer()`, pass `wsManager` as the third argument to `registerRoutes(service, resolvedDir, wsManager)` so the /api/status endpoint has access to client counts.
  </action>
