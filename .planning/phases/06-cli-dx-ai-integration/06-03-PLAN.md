**Structure:**

1. **Header**: "SmartB Diagrams" + one-line description: "AI observability diagrams -- see what your AI is thinking"

2. **Quick Start** (DX-05):
   ```bash
   npm install -g smartb-diagrams
   smartb init
   smartb serve
   ```
   Brief explanation: init creates `.smartb.json` + sample `reasoning.mmd`, serve starts the viewer and opens the browser.

3. **CLI Commands**:
   Document all 4 commands with their options:
   - `smartb init [--dir path] [--force]` — Initialize project
   - `smartb serve [--port N] [--dir path] [--no-open]` — Start viewer
   - `smartb status [--port N]` — Check server status
   - `smartb mcp [--dir path] [--serve] [--port N]` — Start MCP server

4. **MCP Setup** (DX-04, DX-05):
   Three methods, clearly separated:

   **Claude Code** (recommended):
   ```bash
   claude mcp add --transport stdio smartb -- npx -y smartb-diagrams mcp --dir .
   ```
   Or add `.mcp.json` to project root:
   ```json
   {
     "mcpServers": {
       "smartb": {
         "command": "npx",
         "args": ["-y", "smartb-diagrams", "mcp", "--dir", "."]
       }
     }
   }
   ```

   **Claude Desktop**:
   Add to `~/Library/Application Support/Claude/claude_desktop_config.json`:
   ```json
   {
     "mcpServers": {
       "smartb-diagrams": {
         "command": "npx",
         "args": ["-y", "smartb-diagrams", "mcp", "--dir", "/path/to/project"]
       }
     }
   }
   ```

   **With browser viewer** (MCP + HTTP):
   ```bash
   smartb mcp --dir . --serve --port 3333
   ```
   Explain: `--serve` flag starts HTTP+WS server alongside MCP, sharing the same DiagramService.

5. **MCP Tools**:
   Table of 5 tools with brief descriptions:
   | Tool | Description |
   |------|-------------|
   | `update_diagram` | Create or update a .mmd file |
   | `read_flags` | Read developer flags from a diagram |
   | `get_diagram_context` | Get full diagram state (content, flags, statuses) |
   | `update_node_status` | Set node status (ok, problem, in-progress, discarded) |
   | `get_correction_context` | Get structured correction prompt for a flagged node |

6. **AI Diagram Conventions** (AI-01):
   Document the recommended Mermaid conventions for AI reasoning diagrams:
   - Use `flowchart TD` for sequential reasoning, `flowchart LR` for parallel/branching
   - Node IDs: lowercase-hyphenated (e.g., `analyze-requirements`)
   - Node labels: Short action phrases in quotes (e.g., `["Analyze Requirements"]`)
   - Use status annotations to track progress:
     ```
     %% @status nodeId ok          -- step completed (green)
     %% @status nodeId in-progress -- currently working (yellow)
     %% @status nodeId problem     -- encountered issue (red)
     %% @status nodeId discarded   -- abandoned approach (gray)
     ```
   - Developer flags for feedback:
     ```
     %% @flag nodeId "message"
     ```

7. **Example CLAUDE.md Instructions** (AI-04):
   Provide a copyable section that developers can add to their project's CLAUDE.md:
   ```markdown
   ## SmartB Diagrams Integration

   This project uses SmartB Diagrams for AI reasoning observability.

   ### Diagram Updates
   - Use the `update_diagram` MCP tool to create/update .mmd files
   - Use `flowchart TD` for sequential reasoning steps
   - Node IDs should be lowercase-hyphenated (e.g., analyze-input)
   - Update node status as you work: in-progress -> ok or problem

   ### Responding to Flags
   - Check for developer flags using `read_flags` before starting work
   - When flags exist, use `get_correction_context` to get structured correction guidance
   - Address the flag feedback, then update the diagram accordingly

   ### Status Updates
   - Set nodes to `in-progress` when starting a step
   - Set nodes to `ok` when a step completes successfully
   - Set nodes to `problem` when you encounter an issue
   - Set nodes to `discarded` when abandoning an approach
   ```

8. **How It Works**:
   Brief architecture overview (3-4 sentences): AI writes .mmd files via MCP tools, file watcher detects changes, WebSocket broadcasts to browser, developer sees reasoning in real-time and can flag nodes for correction.

9. **Requirements**:
   - Node.js >= 22
   - npm or npx
