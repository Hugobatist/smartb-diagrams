1. **src/mcp/schemas.ts**: Add a new `GetCorrectionContextInput` schema (raw Zod shape, NOT z.object wrapped -- following existing pattern):
   ```
   export const GetCorrectionContextInput = {
     filePath: z.string().describe('Relative path to the .mmd file'),
     nodeId: z.string().describe('ID of the flagged node to get correction context for'),
   };
   ```

2. **src/mcp/tools.ts**: Register a 5th tool `get_correction_context` following the existing tool pattern. Import `GetCorrectionContextInput` from schemas.
   - Description: "Get structured correction context for a flagged diagram node. Returns the flag message, node context (statuses, other flags), the full diagram content, and a natural language instruction for making corrections. Use this when a developer has flagged a node for correction."
   - Handler logic:
     a. Call `service.readDiagram(filePath)` to get the full diagram state
     b. Find the flag matching `nodeId` in `diagram.flags` (it's a Map keyed by nodeId)
     c. If no flag found on that nodeId, return `isError: true` with message: `No flag found on node "${nodeId}" in ${filePath}`
     d. If flag found, build a CorrectionContext object:
        ```json
        {
          "correction": {
            "nodeId": "<flagged node>",
            "flagMessage": "<flag message>"
          },
          "diagramState": {
            "filePath": "<path>",
            "mermaidContent": "<full Mermaid content>",
            "allFlags": [{ "nodeId": "...", "message": "..." }],
            "statuses": { "<nodeId>": "<status>", ... }
          },
          "instruction": "The developer flagged node \"<nodeId>\" with the message: \"<message>\". Review the diagram and update it to address this feedback. Use the update_diagram tool to write the corrected Mermaid content."
        }
        ```
     e. Return `{ content: [{ type: 'text', text: JSON.stringify(context, null, 2) }] }`
   - Wrap in try/catch like existing tools, return `isError: true` on exceptions

   Update the JSDoc comment at the top of registerTools to say "5 MCP tools" instead of "4 MCP tools".

3. **test/mcp/tools.test.ts**: Create test file for the get_correction_context tool:

   - Test: calling with a flagged nodeId returns the expected CorrectionContext structure (correction.nodeId, correction.flagMessage, diagramState.mermaidContent, instruction contains the flag message)
   - Test: calling with a non-flagged nodeId returns isError:true
   - Test: calling with a non-existent file returns isError:true
   - Use mkdtemp + writeFile for isolated test fixtures
   - Don't need full MCP server setup -- test the handler logic directly by calling service methods and verifying the context shape. Alternatively, create McpServer, register tools, but the simplest approach is testing the service-level logic.

   Since the tool handler is registered inside registerTools() and not independently exportable, test via the DiagramService interface directly:
   - Read diagram, check flags exist
   - Verify the correction context shape can be constructed from the service output
   - This validates the data layer; the tool registration is validated by typecheck.
