  <action>
Create `src/diagram/graph-types.ts` with all graph model types. Import `Flag`, `NodeStatus`, and `ValidationResult` from `./types.js`.

Define these types exactly:

1. `NodeShape` — union of 13 shapes: `'rect' | 'rounded' | 'stadium' | 'subroutine' | 'cylinder' | 'circle' | 'asymmetric' | 'diamond' | 'hexagon' | 'parallelogram' | 'parallelogram-alt' | 'trapezoid' | 'trapezoid-alt'`

2. `EdgeType` — union of 5 types: `'arrow' | 'open' | 'dotted' | 'thick' | 'invisible'`

3. `FlowDirection` — union: `'TB' | 'LR' | 'BT' | 'RL'`

4. `GraphNode` interface:
   - `id: string` — the node identifier (e.g., `A`, `nodeX`)
   - `label: string` — display text (falls back to id if no label specified)
   - `shape: NodeShape` — defaults to `'rect'`
   - `status?: NodeStatus` — from annotations
   - `flag?: Flag` — from annotations
   - `subgraphId?: string` — which subgraph this node belongs to (undefined for root-level)
   - `cssClass?: string` — inline class assignment via `:::className`
   - `x?: number`, `y?: number`, `width?: number`, `height?: number` — layout fields, populated by Phase 11, undefined in Phase 10

5. `GraphEdge` interface:
   - `id: string` — generated: `${from}->${to}` or `${from}->${to}#${index}` for multi-edges
   - `from: string` — source node/subgraph ID
   - `to: string` — target node/subgraph ID
   - `label?: string` — optional edge label text
   - `type: EdgeType` — arrow style
   - `bidirectional?: boolean` — for `<-->` edges

6. `GraphSubgraph` interface:
   - `id: string` — subgraph identifier
   - `label: string` — display label (may differ from ID when bracket syntax used)
   - `parentId: string | null` — null for root-level subgraphs
   - `nodeIds: string[]` — directly contained node IDs
   - `childSubgraphIds: string[]` — nested subgraph IDs

7. `GraphModel` interface:
   - `diagramType: 'flowchart' | 'graph'`
   - `direction: FlowDirection`
   - `nodes: Map<string, GraphNode>`
   - `edges: GraphEdge[]`
   - `subgraphs: Map<string, GraphSubgraph>`
   - `classDefs: Map<string, string>` — classDef name -> CSS properties string
   - `nodeStyles: Map<string, string>` — nodeId -> inline style string
   - `linkStyles: Map<number, string>` — edge index -> CSS properties string
   - `classAssignments: Map<string, string>` — nodeId -> classDef name
   - `filePath: string`
   - `flags: Map<string, Flag>`
   - `statuses: Map<string, NodeStatus>`
   - `validation: ValidationResult`

Also export two constant lookup tables (used by parser and serializer):

8. `SHAPE_PATTERNS` — Array of `{ open: string; close: string; shape: NodeShape }` ordered longest-first for correct precedence. Include all 13 shapes with their bracket syntax.

9. `EDGE_SYNTAX` — Record mapping EdgeType to its serialization syntax: `{ arrow: '-->', open: '---', dotted: '-.->', thick: '==>', invisible: '~~~' }`.

Target: ~80-90 lines. File must compile with `npm run typecheck`.
  </action>
