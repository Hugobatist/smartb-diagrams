Create `src/diagram/graph-parser.ts` implementing the multi-pass parser pipeline. The function signature:

```typescript
export function parseMermaidToGraph(rawContent: string, filePath: string): GraphModel
```

**Implementation structure (multi-pass pipeline):**

**Pre-processing:**
- Import `parseFlags`, `parseStatuses`, `stripAnnotations` from `./annotations.js`
- Import `validateMermaidSyntax` from `./validator.js`
- Import `SHAPE_PATTERNS` from `./graph-types.js`
- Parse flags and statuses from raw content
- Strip annotations to get pure Mermaid content

**Pass 1: Parse direction line**
- Match first non-empty non-comment line against `/(flowchart|graph)\s+(TB|TD|BT|LR|RL)/`
- Normalize `TD` to `TB`
- Store `diagramType` and `direction`

**Pass 2: Extract style directives (before removing them)**
- Parse `classDef name styles;` lines -> `classDefs` Map
- Parse `style nodeId styles;` lines -> `nodeStyles` Map
- Parse `linkStyle N styles;` lines -> `linkStyles` Map
- Parse `class nodeA,nodeB className;` lines -> `classAssignments` Map

**Pass 3: Parse subgraph structure**
- Reuse regex patterns from collapser.ts: `SUBGRAPH_START`, `SUBGRAPH_END`
- Build `subgraphs` Map with parentId, nodeIds, childSubgraphIds
- Track which lines are inside which subgraph for node assignment

**Pass 4: Parse node definitions with shapes**
- For each non-directive, non-subgraph, non-comment line, try to extract node definitions
- Use `SHAPE_PATTERNS` array (longest bracket patterns first) to detect shapes
- Handle quoted labels: `["text with (parens)"]` â€” match outermost brackets, strip quotes
- Handle `:::className` suffix: strip it before shape detection, store in classAssignments
- Create `GraphNode` objects in `nodes` Map
- Set `subgraphId` based on which subgraph scope the line appears in

**Pass 5: Parse edge definitions**
- Regex strategy: for each line, try to match edge patterns
- Edge pattern: `nodeA edgeOp nodeB` where edgeOp can include label syntax
- Handle chained edges: `A --> B --> C` -> two edges (A->B, B->C)
- Handle pipe labels: `-->|"text"|` and inline labels: `-- "text" -->`
- Handle bidirectional: `<-->`, `<-.->`, `<==>` -> set `bidirectional: true`
- For each endpoint, create implicit node if not already in `nodes` Map (shape: 'rect', label: id)
- Generate edge IDs: `${from}->${to}` for first occurrence, `${from}->${to}#${N}` for duplicates

**Pass 6: Merge annotations into nodes**
- For each flag in `flags` Map, find matching node and set `node.flag`
- For each status in `statuses` Map, find matching node and set `node.status`

**Pass 7: Validate**
- Run `validateMermaidSyntax(mermaidContent)` and store result

**Return the assembled GraphModel.**

**Critical implementation details:**
- Process lines sequentially, tracking subgraph scope with a stack
- Strip `:::className` from node references BEFORE shape/edge parsing
- Check `SHAPE_PATTERNS` in order (longest open bracket first) for correct precedence
- For edge parsing on a single line, use a loop: parse leftmost edge, then recurse on remainder for chained edges
- When creating implicit nodes from edges, check both `nodes` Map AND `subgraphs` Map â€” edge endpoints that match subgraph IDs are subgraph-to-node edges (still store as regular edge, the `from`/`to` values may be subgraph IDs)

**If the file approaches 450+ lines**, split node-shape parsing into a separate `node-parser.ts` helper. The orchestration function `parseMermaidToGraph()` should stay in `graph-parser.ts`.
