  <action>
1. Create `src/mcp/tools.ts` with a `registerTools(server: McpServer, service: DiagramService)` function that registers all 4 tools using `server.registerTool()`.

   Import schemas from `./schemas.js`. Import types from SDK: `import { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js'`.

   **Tool 1: update_diagram** (MCP-02)
   - Name: `update_diagram`
   - Description: `'Create or update a Mermaid diagram (.mmd file). The content should be valid Mermaid syntax. Changes appear in the browser viewer within 100ms via WebSocket.'`
   - inputSchema: `UpdateDiagramInput` from schemas
   - Handler: Call `service.writeDiagram(filePath, content)`. Return success message with file path. Wrap in try/catch -- if error, return `{ content: [{ type: 'text', text: error.message }], isError: true }`.

   **Tool 2: read_flags** (MCP-03)
   - Name: `read_flags`
   - Description: `'Read all active developer flags from a .mmd diagram file. Flags are annotations added by developers to signal issues or requests to the AI.'`
   - inputSchema: `ReadFlagsInput` from schemas
   - Handler: Call `service.getFlags(filePath)`. Return JSON stringified array of flags with nodeId and message fields. If file not found, return isError: true with helpful message.

   **Tool 3: get_diagram_context** (MCP-04)
   - Name: `get_diagram_context`
   - Description: `'Get the current state of a diagram including Mermaid content, flags, node statuses, and validation info. Use this to understand the diagram before making changes.'`
   - inputSchema: `GetDiagramContextInput` from schemas
   - Handler: Call `service.readDiagram(filePath)`. Build a context object with: mermaidContent, flags (as array), statuses (as object), validation result (valid, errors, diagramType), filePath. Return JSON stringified. On error, return isError: true.

   **Tool 4: update_node_status** (MCP-05)
   - Name: `update_node_status`
   - Description: `'Set the status of a specific node in a diagram. Status values: "ok" (green), "problem" (red), "in-progress" (yellow), "discarded" (gray). Status renders as color in the browser viewer.'`
   - inputSchema: `UpdateNodeStatusInput` from schemas
   - Handler: Call `service.setStatus(filePath, nodeId, status)`. Return success message. On error, return isError: true.

   **Error handling pattern for all tools:**
   ```typescript
   try {
     // ... tool logic
     return { content: [{ type: 'text', text: resultText }] };
   } catch (error) {
     const message = error instanceof Error ? error.message : 'Unknown error';
     return { content: [{ type: 'text', text: message }], isError: true };
   }
   ```

   Never include stack traces in error responses (they leak implementation details to AI agents).

2. Update `src/mcp/server.ts`:

   - Call `registerTools(server, service)` inside `createMcpServer()` after McpServer creation

1. Create `src/mcp/resources.ts` with a `registerResources(server: McpServer, service: DiagramService)` function.

   Import from SDK: `import { McpServer, ResourceTemplate } from '@modelcontextprotocol/sdk/server/mcp.js'`.

   **Resource 1: diagram-list** (MCP-06)
   - Key: `'diagram-list'`
   - URI: `'smartb://diagrams'`
   - Metadata: `{ title: 'Available Diagrams', description: 'List of all .mmd diagram files in the project', mimeType: 'application/json' }`
   - Handler: Call `service.listFiles()`, return `{ contents: [{ uri: uri.href, mimeType: 'application/json', text: JSON.stringify({ files }) }] }`

   **Resource 2: diagram-content** (MCP-07)
   - Key: `'diagram-content'`
   - URI: `new ResourceTemplate('smartb://diagrams/{filePath}', { list: async () => { ... } })`
   - The `list` callback calls `service.listFiles()` and returns `{ resources: files.map(f => ({ uri: \`smartb://diagrams/\${encodeURIComponent(f)}\`, name: f })) }`
   - Metadata: `{ title: 'Diagram Content', description: 'Content of a specific .mmd diagram file', mimeType: 'text/plain' }`
   - Handler: Decode `filePath` from URI params, call `service.readDiagram(decoded)`, return the mermaidContent as text/plain. On error, return empty contents array (resources don't have isError).

   Use `decodeURIComponent()` on the filePath parameter since file paths with special characters will be URL-encoded.

2. Update `src/mcp/server.ts`:
