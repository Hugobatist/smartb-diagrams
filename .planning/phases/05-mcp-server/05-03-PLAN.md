  <name>Task 1: Add --serve mode for shared MCP+HTTP process with graceful shutdown</name>
  <files>
    src/mcp/server.ts
    src/cli.ts
    src/server/server.ts
  </files>
  <action>
1. Update `src/mcp/server.ts` -- modify `startMcpServer()` to accept an options object:

   ```typescript
   interface McpServerOptions {
     dir: string;
     serve?: boolean;
     port?: number;
   }
   ```

   When `serve` is true:
   - Import `createHttpServer` from `../server/server.js` and `detect` from `detect-port`
   - Create the DiagramService FIRST (single instance)
   - Pass the SAME DiagramService to BOTH `createMcpServer(service)` and use it in the HTTP server

   The challenge: `createHttpServer()` currently creates its own DiagramService internally. To share the instance, refactor `createHttpServer()` in server.ts to optionally accept an existing DiagramService. Add an optional `service` parameter:
   ```typescript
   export function createHttpServer(projectDir: string, existingService?: DiagramService): ServerInstance
   ```
   If `existingService` is provided, use it instead of creating a new one. This is a minimal, backward-compatible change.

   When `serve` is true in startMcpServer:
   - Create DiagramService
   - Create MCP server with createMcpServer(service)
   - Create HTTP server with createHttpServer(projectDir, service)
   - Detect available port (default 3333)
   - Start HTTP server listening
   - Connect MCP server to StdioServerTransport
   - Log the HTTP URL to stderr

   When `serve` is false (default -- stdio-only):
   - Create DiagramService
   - Create MCP server with createMcpServer(service)
   - Connect to StdioServerTransport
   - No HTTP server started (lightweight for AI tools that don't need the browser)

2. Implement graceful shutdown in `startMcpServer()`:
   - Listen for `process.stdin` `end` event (parent process disconnected from stdio)
   - Listen for `SIGINT` and `SIGTERM`
   - On shutdown: if HTTP server is running, close fileWatcher, wsManager, httpServer in order. Then `process.exit(0)`.
   - Use the same shutdown order as the existing `startServer()`: fileWatcher.close() -> wsManager.close() -> httpServer.close()

3. Update `src/cli.ts` -- add `--serve` and `--port` options to the `mcp` command:
   ```typescript
   program
     .command('mcp')
     .description('Start the MCP server for AI tool integration (stdio transport)')
     .option('-d, --dir <path>', 'project directory', '.')
     .option('-s, --serve', 'also start HTTP+WS server for browser viewing')
     .option('-p, --port <number>', 'HTTP server port (requires --serve)', '3333')
     .action(async (options: { dir: string; serve?: boolean; port: string }) => {
       const { startMcpServer } = await import('./mcp/server.js');
       await startMcpServer({
         dir: options.dir,
         serve: options.serve,
         port: parseInt(options.port, 10),
       });
     });
   ```

4. Update `src/server/server.ts` -- add the optional `existingService` parameter to `createHttpServer()`:
   ```typescript
   export function createHttpServer(projectDir: string, existingService?: DiagramService): ServerInstance {
     const resolvedDir = path.resolve(projectDir);
     const service = existingService ?? new DiagramService(resolvedDir);
     // ... rest unchanged
   }
   ```
   This is a 1-line change: replace `const service = new DiagramService(resolvedDir)` with `const service = existingService ?? new DiagramService(resolvedDir)`. All existing callers (startServer, tests) continue to work unchanged.
  </action>
