**clipboard.js** (~100 lines):
Create an IIFE module exposing `window.SmartBClipboard` with the following API:

Internal state: `var buffer = null;` -- stores `{ nodeId, label }` or null.

Functions:
- `copy()`:
  1. Get selection via `SmartBSelection.getSelected()`
  2. If no selection or selection type is not 'node', return false
  3. Get editor textarea: `document.getElementById('editor')`
  4. Get node label: `MmdEditor.getNodeText(editor.value, sel.id)`
  5. Store in buffer: `buffer = { nodeId: sel.id, label: label }`
  6. Return true

- `paste()`:
  1. If `!buffer`, return false
  2. Call `MmdEditor.applyEdit(function(c) { var newId = MmdEditor.generateNodeId(c); return MmdEditor.addNode(c, newId, buffer.label + ' (copy)'); })`
  3. Return true

- `duplicate()`:
  1. Get selection via `SmartBSelection.getSelected()`
  2. If no selection or type !== 'node', return false
  3. Call `MmdEditor.applyEdit(function(c) { return MmdEditor.duplicateNode(c, sel.id); })`
  4. Return true

- `hasContent()`: Return `buffer !== null`
- `clear()`: Set `buffer = null`

**Important design notes:**
- Uses internal JS buffer, NOT browser Clipboard API (localhost is not HTTPS)
- `paste()` generates new IDs via `MmdEditor.generateNodeId()` which avoids ID collisions
- `duplicate()` uses existing `MmdEditor.duplicateNode()` which already appends "(copy)"
- All operations guard with `window.SmartBSelection`, `window.MmdEditor` checks
- Uses `var` for codebase consistency

**app-init.js keyboard shortcut additions:**

In the main keyboard handler (line 96, `document.addEventListener('keydown', function(e) {`), add these shortcuts. The handler already returns early if `e.target === editor` (line 98), `e.target.closest('.flag-popover')` (line 99), or `e.target.closest('.search-bar')` (line 100). New shortcuts go inside this guarded block.

1. **Ctrl+Shift+Z (Redo)** -- Add BEFORE the existing Ctrl+Z handler (line 120) since the Shift check must come first:
   ```javascript
   if ((e.key === 'z' || e.key === 'Z') && (e.ctrlKey || e.metaKey) && e.shiftKey) {
       e.preventDefault();
       MmdEditor.redo();
       return;
   }
   ```
   Also add Ctrl+Y as alternative redo:
   ```javascript
   if (e.key === 'y' && (e.ctrlKey || e.metaKey)) {
       e.preventDefault();
       MmdEditor.redo();
       return;
   }
   ```

2. **Ctrl+C (Copy)** -- Add after the zoom shortcuts (line 137):
   ```javascript
   if (e.key === 'c' && (e.ctrlKey || e.metaKey) && !e.shiftKey) {
       if (window.SmartBClipboard && SmartBClipboard.copy()) {
           e.preventDefault();
           if (window.toast) toast('Nodo copiado');
       }
       return;
   }
   ```
   **Important:** Only preventDefault if copy() returned true (a node was actually copied). If no node is selected, let the event propagate normally so browser text copy still works on other elements.

3. **Ctrl+V (Paste)** -- Add after Ctrl+C:
   ```javascript
   if (e.key === 'v' && (e.ctrlKey || e.metaKey) && !e.shiftKey) {
       if (window.SmartBClipboard && SmartBClipboard.hasContent()) {
           e.preventDefault();
           SmartBClipboard.paste();
           if (window.toast) toast('Nodo colado');
       }
       return;
   }
   ```

4. **Ctrl+D (Duplicate)** -- Add after Ctrl+V:
   ```javascript
   if (e.key === 'd' && (e.ctrlKey || e.metaKey)) {
       e.preventDefault();
       if (window.SmartBClipboard && SmartBClipboard.duplicate()) {
           if (window.toast) toast('Nodo duplicado');
       }
       return;
   }
   ```
   Always preventDefault for Ctrl+D to avoid browser bookmark dialog.

5. **Also check for contenteditable** -- The existing guard checks `e.target === editor` but should also guard against contenteditable elements (inline edit overlay). Add after line 98:
   ```javascript
   if (e.target.getAttribute && e.target.getAttribute('contenteditable') === 'true') return;
   ```
