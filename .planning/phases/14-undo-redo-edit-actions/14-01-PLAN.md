  <action>
**command-history.js** (~80 lines):
Create an IIFE module exposing `window.SmartBCommandHistory` with the following API:
- `execute(command)` -- Push command `{ before, after, description }` to undoStack. Cap at MAX_HISTORY=100 entries (shift oldest). Clear redoStack on new edit.
- `undo()` -- Pop from undoStack, push to redoStack, return `command.before` content string. Return null if empty.
- `redo()` -- Pop from redoStack, push to undoStack, return `command.after` content string. Return null if empty.
- `canUndo()` -- Returns boolean (undoStack.length > 0).
- `canRedo()` -- Returns boolean (redoStack.length > 0).
- `clear()` -- Empty both stacks. Called on file switch.
- `getUndoCount()` -- Returns undoStack.length.
- `getRedoCount()` -- Returns redoStack.length.

Both stacks are plain arrays. Use `var` for consistency with existing codebase style (not let/const). Emit `history:changed` via SmartBEventBus after execute/undo/redo/clear (guard with `if (window.SmartBEventBus)`).

**editor-popovers.js** (~130 lines):
Extract these functions from diagram-editor.js into a new IIFE module exposing `window.SmartBEditorPopovers`:
- `closeEditorPopover()` (lines 219-222 of diagram-editor.js)
- `createPopover(x, y)` (lines 224-242)
- `showAddNodePopover(clientX, clientY)` (lines 244-312)
- `showAddEdgePopover(clientX, clientY, fromId, toId)` (lines 314-380)

The extracted module needs access to MmdEditor functions. Use `window.MmdEditor` references directly since diagram-editor.js loads first. Specifically:
- `MmdEditor.generateNodeId(editor.value)` for suggesting IDs
- `MmdEditor.applyEdit(fn)` for creating nodes/edges
- `MmdEditor.addNode(c, id, label)` and `MmdEditor.addEdge(c, from, to, label)` for edit functions
- `DiagramDOM.highlightNode()` for edge source highlighting

Keep `closeEditorPopover` and `createPopover` as internal functions. Expose:
- `showAddNodePopover(clientX, clientY)`
- `showAddEdgePopover(clientX, clientY, fromId, toId)`
- `closePopover()` (public alias for closeEditorPopover)

Both modules use 'use strict' inside IIFE. No external dependencies beyond existing window globals.
